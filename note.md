# Textree — 기능 명세 (Notion 기능 전수 분류)

Notion의 편의 기능을 가능한 한 빠짐없이 나열하고, **파일시스템 기반 단일 사용자 로컬 앱**이라는 제약에 비추어 분류했습니다.

## 분류 범례

- ✅ **그대로 가능** — 파일/폴더만으로 자연스럽게 구현
- 🔧 **변형하면 가능** — frontmatter / 사이드카 파일 / 로컬 규칙으로 재현 (구현 방식 명시)
- ❌ **제외** — SaaS·서버·다중 사용자 본질이라 이식 불가 (문서 맨 끝에 사유와 함께 별도 정리)

> 핵심 전제: **DB 없음 ≠ 데이터베이스 기능 없음.**
> 노션 DB의 "행 = 속성" 모델은 `.md` 파일 + **YAML frontmatter**로 표현하고,
> 뷰(테이블/보드/캘린더/갤러리)는 앱이 런타임에 frontmatter를 인덱싱해 계산합니다.
> 별도 DB 엔진 없이, 파일 자체가 레코드입니다.

## 1. 에디터 & 블록

### 텍스트 / 서식
- ✅ 문단, 제목 H1–H4
- ✅ 굵게 / 기울임 / 밑줄 / 취소선 / 인라인 코드
- ✅ 글자 색·배경(하이라이트) — `==..==` 또는 HTML/span 보존
- ✅ 마크다운 단축 입력 (`#`, `-`, `>`, `` ``` `` 자동 변환)
- ✅ 슬래시 커맨드(`/`)로 블록 삽입
- ✅ "Turn into" (블록 타입 변환)

### 블록 종류
- ✅ 불릿/번호 리스트
- ✅ 체크박스(To-do) — `- [ ]` / `- [x]`
- ✅ 토글 리스트 — `<details>` 또는 앱 전용 접기 마크업
- ✅ 인용(Quote), 콜아웃(Callout), 구분선(Divider)
- ✅ 코드 블록 (언어별 신택스 하이라이트)
- ✅ 수식 — 인라인 `$..$`, 블록 `$$..$$` (KaTeX)
- ✅ 단순 테이블 (마크다운 파이프 테이블)
- 🔧 다단 컬럼 레이아웃 — 마크다운 비표준이므로 앱 전용 컨테이너 마크업 또는 HTML로 직렬화
- 🔧 토글 제목(접히는 헤딩) — 앱 내 상태, 파일엔 일반 헤딩 + 접힘 메타
- ✅ 목차(TOC) 블록 — 헤딩에서 자동 생성(렌더 시점)
- 🔧 동기화 블록(Synced block) — 한 곳의 `.md` 조각을 여러 노트에서 transclude(`![[note#^block]]` 류 위키 임베드)로 참조
- 🔧 버튼/템플릿 버튼 — 앱 액션을 frontmatter/사이드카에 정의

### 임베드 & 미디어
- ✅ 이미지 — 로컬 파일 저장 + 상대경로 `![](assets/img.png)`
- ✅ 파일 첨부 — 로컬 저장 + 링크
- ✅ 동영상/오디오 — 로컬 파일 또는 URL 임베드
- ✅ PDF 임베드 — 로컬 파일 뷰어
- ✅ 웹 북마크 / 링크 프리뷰 — URL 메타 캐시를 사이드카에 저장
- 🔧 코드/디자인 임베드(CodePen, Figma, 지도 등) — iframe 임베드(온라인 시) + 원본 URL 보존
- ✅ 이모지 / 아이콘 / 커버 이미지 — frontmatter에 `icon:`, `cover:` 기록

### 블록 조작
- ✅ 드래그 앤 드롭 재정렬
- ✅ 들여쓰기/중첩
- ✅ 블록 단위 색/배경
- 🔧 블록 참조/링크(특정 블록으로 이동) — 블록 앵커 ID 부여(`^id`) 후 위키링크

## 2. 페이지 기능

- ✅ 페이지 아이콘 (이모지/이미지) — frontmatter `icon:`
- ✅ 페이지 커버 — frontmatter `cover:`
- ✅ 하위 페이지(중첩) — **폴더 구조에 직접 매핑** (`폴더명/폴더명.md`)
- 🔧 페이지 링크 & 백링크 — 위키링크 `[[노트]]` 인덱싱으로 양방향 계산
- ✅ 즐겨찾기 / 고정 — `.textree/favorites.json`
- 🔧 페이지 히스토리/버전 관리 — **Git 연동** 또는 로컬 스냅샷(`.textree/history/`)
- ✅ 페이지 복제 — 파일/폴더 복사
- ✅ 이동(Move to) — 파일/폴더 이동
- 🔧 템플릿(페이지 템플릿) — `.textree/templates/*.md`에서 생성
- ✅ 전체 너비 / 작은 텍스트 / 폰트(기본·세리프·모노) — 페이지 뷰 설정(frontmatter)
- 🔧 페이지 잠금 — 앱 내 편집 잠금 플래그(frontmatter)
- ✅ 단어 수 / 페이지 정보 — 렌더 시점 계산

## 3. 네비게이션 & 조직화

- ✅ 좌측 사이드바 트리 — **폴더 트리와 1:1 동기화**
- ✅ 전문 검색(full-text) — 로컬 인덱스(예: tantivy), DB 불필요
- ✅ 빠른 전환(Quick Switcher, Cmd+P/K) — 파일 인덱스 기반
- ✅ 즐겨찾기 / 최근 방문 섹션
- ✅ 브레드크럼 — 파일 경로에서 생성
- ✅ 트리 펼침/접힘, 드래그로 재정렬·중첩(= 파일 이동)
- 🔧 트리 정렬 순서(수동) — `.textree/order.json`
- 🔧 명령 팔레트(Cmd+K) — 앱 전 기능 진입점

## 4. 데이터베이스 (frontmatter 기반) — 핵심

> 폴더 하나 = 데이터베이스, 그 안의 각 `.md` = 행(레코드), frontmatter = 속성.
> 뷰는 데이터를 복제하지 않고 **같은 파일 집합을 다르게 렌더**합니다.

### 속성 타입
- ✅ 텍스트 / 숫자 / 체크박스 / URL / 이메일 / 전화 — frontmatter 스칼라
- ✅ 셀렉트 / 멀티셀렉트 / 상태(Status) — frontmatter 문자열·배열 + 스키마 정의(`.textree/db.json`)
- ✅ 날짜 / 기간 — ISO 8601 문자열
- ✅ 생성일 / 수정일 — 파일 mtime/ctime 또는 frontmatter
- 🔧 사람(Person) — 단일 사용자라 사실상 본인/태그 문자열로 축소
- ✅ 파일 & 미디어 — 상대경로 링크 배열
- 🔧 관계(Relation) — 다른 노트로의 위키링크, 인덱서가 양방향 해석
- 🔧 롤업(Rollup) — 관계를 따라가 집계, 렌더 시점 계산
- 🔧 수식(Formula) — 앱 내 수식 엔진이 frontmatter 값으로 계산(파일엔 식만 저장)
- 🔧 ID(자동 증가) — 앱이 발급, frontmatter에 고정 기록
- 🔧 버튼 속성 — 행 단위 액션 정의

### 뷰
- ✅ 테이블 / 리스트 / 갤러리
- ✅ 보드(칸반) — `status` 속성 기준 그룹핑
- ✅ 캘린더 — `date` 속성 기준
- ✅ 타임라인(간트) — 시작/종료 날짜 기준
- 🔧 차트 뷰 — frontmatter 집계를 로컬 차트로 렌더 *(노션 2026 신규 차트에 대응)*
- ✅ 필터 / 정렬 / 그룹·하위그룹 — 뷰 정의를 `.textree/views/*.json`에 저장
- ✅ 연결된 데이터베이스(linked DB) — 동일 폴더를 다른 노트에서 뷰로 임베드
- 🔧 행 단위 템플릿 — `.textree/templates/`
- ✅ 합계/개수 등 푸터 집계
- 🔧 데이터베이스 자동화(편집 트리거 등) — 로컬 규칙 엔진(파일 워처 + 액션)

## 5. AI 기능

> UJ의 로컬 GPU 인프라(GPUStack/RTX 5090)를 고려하면 로컬 LLM 연동이 자연스럽고,
> 프라이버시 우선 철학과도 맞습니다. API 연동도 선택지.

- 🔧 작성 보조(개선·요약·번역·톤 변경·맞춤법) — 로컬/원격 LLM 호출, 결과를 블록에 삽입
- 🔧 워크스페이스 Q&A(노션 핵심 AI) — **로컬 파일 인덱스 위 RAG**로 재현 (UJ의 FluxIndex류와 정합)
- 🔧 DB 자동 채움(autofill) — 선택 행의 frontmatter를 LLM으로 채움
- 🔧 AI 미팅 노트/전사 — 오디오 → 로컬 STT → `.md` 생성 *(노션 Cmd+K 미팅 노트에 대응)*
- 🔧 AI 블록(문서 내 자동 갱신 요약) — 캐시된 결과 + 재생성 트리거
- 🔧 커스텀 프롬프트 템플릿 — `.textree/prompts/`
- 🔧 코드 실행(Workers 대응) — 로컬 앱은 오히려 샌드박스 코드 실행이 더 쉬움(선택 기능)

## 6. 가져오기 / 내보내기 / 연동

- ✅ 가져오기(Markdown, CSV, HTML, Word, Evernote 등) → `.md` + 첨부로 변환
- ✅ 내보내기 — 이미 `.md`라 사실상 무손실. PDF/HTML 추가 출력 지원
- 🔧 웹 클리퍼(브라우저 확장) — 웹페이지를 vault에 `.md`로 저장
- 🔧 정적 사이트 내보내기 — vault → 정적 HTML(노션 "퍼블리시"의 로컬 대체, 호스팅은 사용자 몫)
- 🔧 캘린더 연동(.ics) — frontmatter 날짜 → `.ics` 내보내기. *(참고: 노션 본체도 네이티브 .ics 동기화는 아직 없음)*
- 🔧 기기 간 동기화 — 앱 내장이 아니라 Dropbox/iCloud/Git/Syncthing에 위임 (파일 기반의 자연스러운 이점)

## 8. 제외 항목 (파일시스템·단일 사용자 본질상 이식 불가)

다음은 "로컬·단일 사용자·서버 없음"이라는 정체성과 정면 충돌하므로 의도적으로 제외합니다.
(대부분은 Textree가 *포기하는 대가로 소유권·이식성·프라이버시를 얻는* 항목입니다.)

- ❌ **실시간 다중 사용자 동시 편집** — 단일 사용자 전제. 비동기 협업은 Git으로 우회.
- ❌ **공유 / 권한 / 게스트 / 팀스페이스 / 멤버 관리** — 접근 제어할 서버가 없음.
- ❌ **알림 인박스(다른 사람의 멘션·댓글 알림)** — 협업의 부산물이라 단독 환경에선 의미 소멸.
- ❌ **댓글·토론(소셜 측면)** — 로컬 사이드카로 *저장*은 가능하나 단독 사용 시 효용 거의 없음 → 기본 제외.
- ❌ **제안 모드(Suggesting / 편집 제안)** — 협업 리뷰 전제.
- ❌ **Notion 퍼블리시 / Notion Sites / 공개 공유 링크 / 커스텀 도메인 / SEO** — 호스팅 SaaS 기능. (단, *정적 사이트 내보내기*는 7번에 포함.)
- ❌ **클라우드 자동 동기화 / 서버 저장** — 파일시스템이 원천이며 동기화는 외부 도구에 위임.
- ❌ **서버형 Workers for Agents(노션 인프라 코드 실행)** — 호스팅 전제. (단, *로컬 코드 실행*은 5번에 대체 포함.)
- ❌ **워크스페이스 분석/감사 로그, 관리자 콘솔, 엔터프라이즈 보안 정책(SSO, AI 제외 영역 등)** — 조직·서버 관리 기능.
- ❌ **공개 템플릿 갤러리(마켓플레이스)** — 커뮤니티 서버 기능. (로컬 템플릿은 2번에 포함.)
- ❌ **Notion 폼(외부 응답 수집)** — 외부인이 제출할 공개 엔드포인트 필요.

## 우선순위 제언

전수 목록은 위와 같고, 구현 난이도·가치를 기준으로 단계를 나누면:

1. **에디터 + 트리 동기화 + 검색** (1~3번의 ✅ 대부분) — 제품의 최소 정체성.
2. **frontmatter 데이터베이스 + 기본 뷰**(테이블/보드/캘린더) — Notion다움의 핵심 차별 기능.
3. **위키링크/백링크 + 관계/롤업** — 지식베이스로의 확장.
4. **로컬 AI(RAG Q&A, 작성 보조)** — UJ의 GPU 인프라와 정합되는 고가치 레이어.
5. **Git 버전 관리·정적 내보내기·웹 클리퍼** 등 주변부.
